参数

◆ vsf_spi_fifo_transfer()

void vsf_spi_fifo_transfer ( vsf_spi_t *  spi_ptr,
void *  out_buffer_ptr,
uint_fast32_t *  out_offset_ptr,
void *  in_buffer_ptr,
uint_fast32_t *  in_offset_ptr,
uint_fast32_t  count 
)

在 SPI 实例上执行基于 FIFO 的数据传输

参数
[in,out]spi_ptr指向结构体 vsf_spi_t 的指针
[in]out_buffer_ptr指向要发送数据的缓冲区指针（仅接收时可为 NULL）
[in,out]out_offset_ptr指向发送缓冲区偏移的指针
[out]in_buffer_ptr指向用于接收数据的缓冲区指针（仅发送时可为 NULL）
[in,out]in_offset_ptr指向接收缓冲区偏移的指针
[in]count要传输的数据单元数（对于仅有命令/地址阶段的 QSPI 操作可以为 0）
注解
数据单元大小由 SPI 模式中配置的数据大小决定（vsf_spi_mode_t）
缓冲区大小要求：4-8 位 = 1 字节，9-16 位 = 2 字节，17-32 位 = 4 字节每个数据单元
在主机模式下，out_buffer_ptr 数据通过 MOSI 引脚发送，in_buffer_ptr 从 MISO 引脚接收
在从机模式下，out_buffer_ptr 数据通过 MISO 引脚发送，in_buffer_ptr 从 MOSI 引脚接收
使用 vsf_spi_ctrl() 配置 QSPI 后，count 参数仅指数据阶段的单元数
空指针规则：out_buffer_ptr 与 in_buffer_ptr 不能同时为 NULL；
  • 若 out_buffer_ptr 为 NULL（仅接收），则 out_offset_ptr 必须为 NULL；
  • 若 in_buffer_ptr 为 NULL（仅读取），则 in_offset_ptr 必须为 NULL；
  • 若某个缓冲区指针非 NULL，其对应偏移指针也必须非 NULL。
驱动实现者应检查参数合法性， 并使用断言（例如 VSF_HAL_ASSERT）处理非法参数。

◆ vsf_spi_request_transfer()

vsf_err_t vsf_spi_request_transfer ( vsf_spi_t *  spi_ptr,
void *  out_buffer_ptr,
void *  in_buffer_ptr,
uint_fast32_t  count 
)

请求 SPI 数据传输操作

参数
[in,out]spi_ptr指向 SPI 实例结构体 vsf_spi_t 的指针
[in]out_buffer_ptr指向要发送数据的缓冲区指针（仅接收时为 NULL）
[out]in_buffer_ptr指向用于接收数据的缓冲区指针（仅发送时为 NULL）
[in]count要传输的数据单元数量（对于仅有命令/地址阶段的 QSPI 操作可以为 0）
返回
vsf_err_t: 如果传输成功启动则返回 VSF_ERR_NONE，否则返回错误码
注解
这是一个异步操作，注册完成中断以获得通知
数据单元大小由 SPI 模式中配置的数据大小决定（vsf_spi_mode_t）
缓冲区大小要求：4-8 位 = 1 字节，9-16 位 = 2 字节，17-32 位 = 4 字节每个数据单元
如果 out_buffer_ptr 和 in_buffer_ptr 都不为 NULL，则以全双工模式交换数据
在主机模式下，out_buffer_ptr 数据通过 MOSI 引脚发送，in_buffer_ptr 从 MISO 引脚接收
在从机模式下，out_buffer_ptr 数据通过 MISO 引脚发送，in_buffer_ptr 从 MOSI 引脚接收
对于调用 vsf_spi_ctrl() 后的 QSPI 操作，此函数将执行所有已配置的阶段（命令、地址、空闲、数据）， count 参数仅指数据阶段的单元数。
空指针规则：out_buffer_ptr 与 in_buffer_ptr 不能同时为 NULL；若某个缓冲区指针为 NULL， 则其对应的偏移指针（如适用）也必须为 NULL；若某个缓冲区指针非 NULL，则其对应偏移指针必须非 NULL。
