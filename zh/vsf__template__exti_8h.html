VSF_EXTI_CFG_PREFIX 是用来配置调用 vsf_exti_*() 的时候实际调用的 API。 例如，当 VSF_EXTI_CFG_PREFIX 配置成 vsf_hw 的时候，调用 vsf_exti_enable() 实际调用的是 vsf_hw_exti_enable()

#undef VSF_EXTI_CFG_PREFIX
#define VSF_EXTI_CFG_PREFIX vsf_example

◆ vsf_exti_mode_t

typedef enum vsf_exti_mode_t vsf_exti_mode_t

预定义的 VSF EXTI 模式，可以在具体的 hal 驱动重新实现。

即使硬件不支持这些功能，但是这些模式是必须实现的。 如果硬件驱动里我们添加了新的模式，那么也需要定义宏 VSF_EXTI_MODE_MASK，它的值是所有模式的值的或。

  • Trigger mode
    • VSF_EXTI_MODE_NONE
    • VSF_EXTI_MODE_EDGE_RISING
    • VSF_EXTI_MODE_EDGE_FALLING
    • VSF_EXTI_MODE_EDGE_RISING_FALLING

可选特性需要提供一个或者多个枚举选项，还需要提供同名的宏，方便用户在运行时判断是否支持。 如果它特性支持多个选项，建议提供对应的 MASK 选项，方便用户在运行时切换到不同的模式。

vsf_exti_channel_cfg_t cfg = {
.mode = VSF_EXTI_EDGE_RISING_FALLING,
...
};
vsf_exti_channel_mask channel_mask = 0x00000001;
vsf_exti_config_channels(&vsf_hw_exti0, channel_mask, &cfg);

预定义的 VSF EXTI 控制命令，可以在特定的 HAL 驱动中重新实现。

注解
作为 vsf_exti_ctrl(vsf_exti_t *exti_ptr, vsf_exti_ctrl_t ctrl, void *param) 函数中的 'ctrl' 参数（第二个参数）
'param' 参数（第三个参数）的类型取决于具体的控制命令

可选控制命令需要提供一个或多个枚举选项，还需要提供同名的宏，以便用户在运行时判断是否支持。 如果该功能支持多个选项，建议提供相应的 MASK 选项，以便用户在编译时检查支持的功能。

功能分类：

  • 必选：无（默认仅定义 __VSF_EXTI_CTRL_DUMMY）
  • 标准可选：无（未定义标准可选功能）

◆ vsf_exti_ctrl_t

enum vsf_exti_ctrl_t

预定义的 VSF EXTI 控制命令，可以在特定的 HAL 驱动中重新实现。

注解
作为 vsf_exti_ctrl(vsf_exti_t *exti_ptr, vsf_exti_ctrl_t ctrl, void *param) 函数中的 'ctrl' 参数（第二个参数）
'param' 参数（第三个参数）的类型取决于具体的控制命令

可选控制命令需要提供一个或多个枚举选项，还需要提供同名的宏，以便用户在运行时判断是否支持。 如果该功能支持多个选项，建议提供相应的 MASK 选项，以便用户在编译时检查支持的功能。

功能分类：

  • 必选：无（默认仅定义 __VSF_EXTI_CTRL_DUMMY）
  • 标准可选：无（未定义标准可选功能）
枚举值
__VSF_EXTI_CTRL_DUMMY 

编译占位值，当没有定义实际控制命令时需要。

注解
此值仅在以下情况需要：使用模板默认枚举定义 (VSF_EXTI_CFG_REIMPLEMENT_TYPE_CTRL == DISABLED) 且所有可选控制命令都被注释掉时。 它确保枚举至少有一个成员，以避免某些不允许空枚举的 C 编译器报错。
如果启用了以下任何控制命令（取消注释），或者在特定硬件驱动中重新定义了枚举 (VSF_EXTI_CFG_REIMPLEMENT_TYPE_CTRL == ENABLED)，只要定义了至少一个实际命令， 就可以删除此 DUMMY 值。
