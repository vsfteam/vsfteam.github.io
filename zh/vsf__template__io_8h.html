类型定义说明

◆ vsf_io_mode_t

typedef enum vsf_io_mode_t vsf_io_mode_t

预定义的 VSF IO 模式，可以在具体的 HAL 驱动重新实现。

即使硬件不支持这些功能，但是这些模式是必须实现的。如果硬件支持更多模式，例如复用推挽输出或者 复用开漏输出，我们可以在硬件驱动里实现它，但是 VSF_IO_AF 模式必须保留。 如果硬件驱动里我们添加了新的模式，那么也需要定义宏 VSF_IO_MODE_MASK，它的值是所有模式的值的或。

  • INPUT/OUTPUT
    • VSF_IO_INPUT
    • VSF_IO_ANALOG
    • VSF_IO_OUTPUT_PUSH_PULL
    • VSF_IO_OUTPUT_OPEN_DRAIN
    • VSF_IO_AF
    • VSF_IO_EXTI
  • PULL_UP_AND_DOWN:
    • VSF_IO_NO_PULL_UP_DOWN
    • VSF_IO_PULL_UP
    • VSF_IO_PULL_DOWN

可选特性需要提供一个或者多个枚举选项，还需要提供同名的宏，方便用户在运行时判断是否支持。 如果它特性支持多个选项，建议提供对应的 MASK 选项，方便用户在运行时切换到不同的模式。

vsf_io_cfg_t cfg = {
{ VSF_PA0, 0, VSF_IO_OUTPUT_PUSH_PULL },
{ VSF_PA1, 0, VSF_IO_ANALOG },
};
vsf_io_config(&vsf_hw_io0, &cfg, 1);
